5.2 Tokens

OZ++ language has the following kinds of tokens:

  • Comments
  • Identifiers
  • Keywords
  • Literals (integers, characters, floating point numbers, character strings)
  • Operators
  • Delimiters

White spaces (i.e. spaces, tabs, newline, newpage, etc) are ignored except as they separate tokens.

When reading a token, the longest possible character string is chosen.

5.2.1 Comments

OZ++ language has two types of comments:

  • Begins with /*, and ends with */ .

    This style can not be nested.

  • Begins with //, and ends with newline codes.

Comments are treated as white spaces.

5.2.2 Identifier

This is a string of numbers (or underscores) that begins with alphabetical character. underscore. Lower case and upper case are distinguished as different character. Furthermore, strings of up to 31 characters are distinguished. Distinction of strings of more than 31 characters can not be guaranteed.

In addition, an identifier of 16 characters containing two underscores followed by [0-9a-fA-F] is distinguished as a class name expressed by its class ID. 

__0001000002000101  // expresses the class ID of 0001000002000101

5.2.4 Literals

OZ ++ language has the following types of literals:

  • Integer constants

    An integer constant is a string of numbers.

    • If a string begins with 0, it represents an octal number. Such strings do not contain the numbers 8 or 9.

    • If a string with 0x or 0X, it represents a hexa-decimal number. Such strings contain (both lower case and upper case) letters from 'a' to 'f'. They represent from 10 to 16, respectively.

    • Otherwise, the string represents a decimal

    If u or U is appended to the string, it represents an unsigned number, if l, or L is appended to the string, it represents a long number, and if ll, lL, LL, or Ll is appended to the string, it represents a long long number. 

    123    // the decimal 123 of int type
0173   // the octal representation of the decimal 123
0x76   // the hexa-decimal representation of the decimal 123
123U   // the decimal 123 of unsigned int type

  • Character Constants

    A character constant is expressed as a single character enclosed by single quotations. Some non-graphic characters, single quotations, etc, can be expressed as escape sequences as shown in Table 5.2--2.

     

    Table 5.1--2:Escape Sequence
    newline NLL (LF)\n
    horizontal tab HT \t
    vertical tab VT \v
    backspace BS \b
    carriage returnCR \r
    formfeed FF \f
    audible alert BEL \a
    backslash \ \\
    question mark ? \?
    single quote ' \'
    double quote " \"
    octal number ooo \ooo
    hex number hh \xhh

    A character constant has the type char, and its value is the number in the character set of the machine. 

    'a' '\n'        // the character a, and the newline code

  • Floating Point Constants

    A floating point number has an integer part, a decimal point, a decimal fraction part, an exponential part after e or E, and a suffix. The suffix may be

    • f or F

    If this suffix

    • is not present, the number is of double type

    • is f or F, the number is of float type 

    1.23  1.23E-1       // double type
1.23F 1.23E-1F      // float type

  • String Literals

    Character strings are array derived types of the char type (see section 5.7). A character string is a sequence of characters enclosed by double quotations. Like character constants, non graphic characters can be expressed by escape sequences. Newline codes must not be present in a character string.

    Adjacent string literals are concatenated, and considered as a single one. 

    "OZ++"
"OZ" "++"         // as "OZ++"

  • User-Defined Constants

    It is possible to declare and use constants inside a shared (c.f. Section 5.15.2).

    User-defined constants have the declared types. 

    float Pie = 3.14;        // constant of float type
int MaxSize = 100;       // constant of int type
char Name = "MyName";    // constant of array 
                         // derived type of the char type
