3.4 Inspector operations

NAME

inspect - inspector

FORMAT

inspect [Local Executor ID]

FUNCTIONS

The function of this tool is to display the contents of process stacks or global objects of the specified executor. It is possible to inspect the global object of the executor or the process stack if the executor is not operating on the same WS. This function uses the executor specified as the Local Executor ID for the address solution. It is also possible to display the contents of stack process on multiple executors. Windows opened by this command allow for users to receive detailed information of items for inspection merely by pointing then double clicking their choice (the function is not available if the window does not open). The following are the main types of windows that can be opened.

  • Debugger
    Debugger window will appear first when using the inspect command. Here the executor ID can be specified for the address solution of an executor ID under inspection. The functions of each button in the window are as follows:

    Button Functions
    Object List Displays the Global Object List window.
    Process List Displays the Process List window.
    Quit Close all windows and exit the executor

  • Global Object List
    his window appears when the Object List button in the debugger window is pressed. This window displays listing of global objects in the executor. The functions of each button in this window are as follows[+]:

    Button Functions
    Refresh Updates displayed information
    Inspect Displays the Global Object Inspect window of selected global object. This function can also be used to suspend the global object status.
    Snap Temporarily saves the instance variable data of selected global object then opens the Global Object Inspect window. The global object will not be suspended while the Global Object Inspect window is open.
    Status Displays the status of the selected global object kept by the object manager.
    Close Closes the Global Object List window.

  • Process List
    This window appears when the Process List button in the debugger window is pressed. This window displays the process list starting with the specified executor in the Debugger window. The functions of each button in this window are as follows[+]:

    Button Functions
    Refresh Updates displayed information
    Inspect Displays the Process Inspect window of the selected process. This function can also be used to suspend processing.
    Kill Terminate selected process.
    Status Displays the execution condition of the selected process.
    Close Closes the Process List window.

  • Object Inspect
    This window is used to display the object content. The function of each button in this window is as follows:

    Button Functions
    Refresh Updates displayed information.
    Inspect Displays the Object Inspect window of the selected object.
    Process This function refers to the global object cell and displays listing of the process by the executor.
    Close Closes the Object Inspect window. However, this function will close all windows opened with the Global Object Inspect window. If the global object has been suspended, this condition will be canceled.

  • Process Inspect
    This window displays the contents of the process stack. The function of each b utton in this window is as follows[+]:。

    Button Functions
    Inspect Displays the Object Inspect window of selected object.
    Kill Terminates concerned process.
    Close Closes the Object Inspect window. If the process execution has been suspended, this condition will be canceled.

Environment variables

OZHOME
Home directory of OZ++

OZSITE_ID
Site ID

OZCLASSPATH
Class directory

Caution:

  • When specifying argument of Local Executor ID with the use of hexadecimal, it is necessary to place 0x at the beginning.
  • The cell (global object) belonging to the object will be placed on hold while displaying object contents. However, this suspension is unrelated to the object manager.
  • The process will be placed on hold while displaying process stacks. As a result, it may influence other processes (exclusive control, etc.).
  • The inspector presumes that the OZ++ system is functioning properly. Therefore, if the executor itself is paused *2, the inspector waits for the restart of the executor.

e.g.

inspect 0x101
This activates the inspector to solve the address of the executor of 101 in hexadecimal.
