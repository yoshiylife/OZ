5.9 Expressions

exp
        : exp_no_comma
        | exp_comma
        ;

The following expressions are available:

  • Primary Expressions
  • Unary Expressions
  • Binary Expressions
  • Condition Expressions
  • Comma Expressions

5.9.1 Primary Expressions

  

exp_no_comma
        | exp_var 
        | exp '.' IDENTIFIER
        | exp_value 
        | '(' exp ')' 
        | exp '->' IDENTIFIER '(' arg_list ')'
        | IDENTIFIER '(' arg_list ')' 
        | exp '.' IDENTIFIER '(' arg_list ')' 
        | exp '=>' IDENTIFIER '(' arg_list ')' inst_option
        | exp '++'
        | exp '--'

arg_list
        : /* empty */
        | exp_no_comma
        | exp_comma
        ;

The following primary expressions are available:

  • Identifiers
  • Literals
  • Expressions Enclosed in Parentheses
  • Message Expressions
  • Instance Generation
  • Postfix Increments / Decrements

5.9.2 Message Expressions

Method expressions is used to activate operators of records and methods of ordinary classes as well as static classes. The description of methods differ from that of operators in the following way:

  • Method 

            | exp '->' IDENTIFIER '(' arg_list ')'

  • Operator 

            | exp '.' IDENTIFIER '(' arg_list ')'


// class A
// record B

A a;
B b;

a->m1();
b.m1();

5.9.3 Generation of Instances

exp_no_comma
        | exp '=>' IDENTIFIER '(' arg_list ')' inst_option

inst_option
        : /* empty */
        | '@' exp 
        ;

This statement is used to generate instances of ordinary classes and static classes, and to activate constructors. 

class A;

A a;

a=>m1();

When instantiating global objects, the location for generation can be specified (see Chapter 5.18).

5.9.4 Unary Expressions

  

exp_no_comma
        | 'fork' exp
        | 'join' exp
        | '++' exp 
        | '--' exp 
        | '!' exp 
        | '~' exp 
        | '+' exp 
        | '-' exp 
        | 'length' exp

Unary expressions are those which perform unary operations in primary expressions.

The following types of unary expressions are available:

  • Generation of Processes (see Section 5.8.3)
  • Joining of Processes (see Section 5.8.3)
  • Arithmetic Operations
  • Increments/Decrements
  • Logical Negations
  • Bitwise Complement
  • Array Lengths (see Chapter 5.7)

The following types of binary operations are available:

  • Multiplicative Operation
  • Additive Operation
  • Shift Operation
  • Relational Operation
  • Bitwise Operation
  • Conditional Operation
  • Assignment

5.9.6 Condition Expressions

exp_no_comma
        | exp '?' exp ':' exp

If the condition is satisfied, the expression following ? is evaluated. Otherwise, the expression following : is evaluated. The value of the expression is considered to be the value of the condition expression.

5.9.7 Comma Expressions

exp_comma
        : exp ',' exp
        ;

A series of expressions separated by commas is called a comma expression. This means that the expressions are evaluated from left to right. The last expression is considered to be the value of the comma expression.
