5.3 Scopes and Blocks

5.3.1 Blocks

Blocks are used to allow compound statements in places where only one statement is written, and vice versa, i.e. a single statement can be used where only one is required (see Chapter 5.10).

5.3.2 Scopes

Scope in OZ++ language is lexical scope, it is determined statically during compilation.

There are four types of scopes: local, class, static class, and record.

  • Local Scope
    A name declared inside a block is local to that block. The name is valid only inside the block.

    Names of call by value parameters of operators and methods are, however, treated as those names that are declared in the outermost block.

    Only names of variables can be declared in a block.

  • Class Scope
    This refers to the scope of a class. The user-defined type (see Chapter 5.5) used here is local to the scope.

    The names of class members are also local to this class, and can be used in the following places:

    1. Inside the method of the class

    2. In case of a method, below the operator -> applied to objects of the class or child classes inheriting the class

    3. Inside the methods of the child classes inheriting the class.

    To make 2 and 3 possible, appropriate access control must be made (see Chapter 5.13).

  • Static Class Scope

    This refers to the scope of a static class. The user-defined type (see Chapter 5.5) used here is local to the scope.

    The names of static class members are also local to the class, and can be used in the following places:

    1. Inside the the method of the static class

    2. In case of a method, below the operator -> applied to objects of the static class

    To make 2 possible, appropriate access control must be performed (see Chapter 5.14).

  • Record Scope
    This refers to the scope of a record. The user-defined type (see Chapter 5.5) used here is local to the scope.

    The names of record members are also local to this record, and can be used in the following places:

    1. Inside the operator of this record

    2. After the operator . applied to the objects of this record

Names are sometimes hidden by the scope which contains them. 

class D : A {
  public: m1;

  int x, y;
  B b;
  C c;

  void m1 (int x)
    {
      int y, i;        

      y = x;
      for (i = 0; i < y; i++)
       {
          int x;

          x +=  y * i;
       }
    }        
}

In the above example, A, B, C, and D are four user-defined types which type names are local to the scope of class D.

In method m1, y is declared locally, and x is declared as a argument. Thus, the instance variables x and y become hidden. Also, as x is locally declared in for statement, the instance variable x of method becomes hidden. If local declarations become hidden in this way, they can not be accessed.
