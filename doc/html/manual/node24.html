5.4 Basic Data Types

 

OZ++ language has the following types of basic data types.

  • Character Type ( char)

    Unsigned can also be specified.

  • Integer Type ( short, int, long, long long)

    This represents the type for integral number. Three types of different sized integers, short, int, long, and long long, can be used. Unsigned can also be specified for each.

  • Floating Point Number Type ( float, double)

    This represents the type for floating point numbers. Two types of different sized floating point numbers can be used, i.e. float and double.

  • Condition Variable Type ( condition)

    This type represents condition variables used to synchronize threads (see Chapter 5.19).

Basic data types are interpreted as values. Assignments and comparisons following the types are executed against the values. In addition, when basic data types are used for assignments, return values, or call by value arguments of method calls after the types, the following events take place:

  • For condition type

    The reference is passed on.

  • Otherwise

    All the values are copied. 

int a, b;

a = 2;            // 2 is copied into a
b = a;            // the value of a is copied into b 
if (a < 10)       // the value of a (i.e. 2) is compared
  {               // to the value 10 
    math->log(2); // the value of a (i.e. 2) is copied, and
                  // substituted into the call by value argument
  }

In OZ++ language, the space consumed by basic data types is not implementation dependent as in C and C++ languages, but is fixed. The alignments of the basic types depend on the execution environment.

5.4.2 0 (Zero)

 

0 can be used against all types in OZ++ language. As shown below, the meaning of 0 is determined by the data type where it is used. 

class A;

int i = 0;     // This means the integer 0.

A a;
if (a == 0)    // This shows that no reference is made
               // to any instances of the class
  ...

  • For basic data types, except the type condition, this shows the value represented by 0 in that type.

  • For condition type, this shows that no reference is made.

  • For class types and static class types, this means that there is no bind for the variable to instance (see Chapters 5.13 and 5.14).

  • For record types, this shows that all members of the record are 0 (see Chapter 5.12).

  • For array types, this means that there is no bind for the variable to instance of array (see Chapter 5.7).

  • For process derived types, this means that there is no bind for the variable to any processes (see Chapter 5.8).
