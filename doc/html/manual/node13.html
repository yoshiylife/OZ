3.2 Workbench operations

3.2.1 Outline

WB performs the following functions:

  • Browses or corrects a school
  • Browses the interface, the source codes, and the configuration of a class
  • Compiles source codes

3.2.2 How to Activate the Work Bench

Generate and launch objects of the class WorkbenchLaunchable.

If activated for the first time, the window for initial setup appears on the screen. Input the class object name. [+]

3.2.3 Functions that are common to both WB and tools which can be activated from WB

  • The button to the extreme left

    • Close (excluding WB)
      Closes windows
    • Open (CFE and SB only)
      Opens a different tool for a new process.
      Select the school from the list given in WB (mentioned later)
    • Preference...
      Performs initial setups (such as Class Object Name, Current Directory, Language)
    • Quit
      Terminates WB

  • The Menu Button of the Window
    Displays the list of all the present existing windows.
    If selected, the window can be either deiconified or raised.

3.2.6 Functions of SB

This is an improved version of a simple browser. It has the following new functions:

  • All classes in the school are displayed in the list.
    f a class is double-clicked, the information concerning the ID and the type (in the file class/staticclass/record/shared/abstract class) is displayed.
    If the 'Edit' button is pressed, editing becomes possible. After editing, the 'Save' button must be pressed to affect the result in the school. Press the 'Done' button to return to the normal browser state.
  • The Package button is used to create a package to be released to the catalog. Click the Package button. A dialog box with package contents will appear after several seconds. At the top of the dialog box is the class to be registered to the school of the package. At the present, there is only one class (future library package will allow multiple classes). The bottom of the dialog box displays the list of possible classes for creation while activating the class indicated at the top. The Now button at the bottom creates a package that specifies a configuration of the class currently under use (with in the cache of the object manager). The Default button creates a package that does not specify configuration but uses the default item specified in the class. Click the Now or Default button to open the catalog browser and to select the directory for registering the created package. When clicking Select at this time without making a selection, the currently displayed directory will be used.
  • The titles 'Public', 'Protected', 'Implementation', and 'Configuration' are in the form of buttons. If clicked, the class information concerning the respective title can be browsed.

3.2.7 Browsing the Class Information

This is a simple browser with a few alterations made to improve its display. Its new function is that it can be connected to a school browser.

  • Window Displaying Interface Information
    After selecting a version ID or a class name, if any, in the displayed list of new classes to the left or in the displayed part of detailed information of a member is clicked while pressing the control button, information concerning the version ID and the type of class is displayed.
    • Operation 

      Search...
        Enables a search for a member

  • Window Displaying Source Codes
    If a class name is clicked while holding the control key down after selecting 'class name', the information concerning the version ID and the type of class is displayed.
    • Operation 

      Search...
        Searches a character string

Goto...
        Displays the line specified by its line number

  • Window Displaying Source Codes
    If a class name is clicked while holding the control key down after selecting 'class name', the information concerning the version ID and the type of class is displayed.
