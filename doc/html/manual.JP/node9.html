2.5 エクゼキューターの説明

2.5.1 executor の起動時のオプションと引数

使い方　

Usage: executor [options] <executor id>
< executor id >スタンドアローンで実行する場合のエグゼキュータＩＤ

options

-H < mbyte ［k］ >
オブジェクトのヒープとして利用する メモリサイズの上限を指定する。 k を付けるとキロバイト指定となる。 （例 1024k は１メガバイト） （デフォルト 16メガバイト(1024*1024単位)）

-M <max>
同時に生成と実行ができるスレッドの最大数を指定する。 （デファルト 256）

-S <site id>
スタンドアローンで実行する場合のサイトＩＤを指定する。 （デフォルト 1）

-T <ticks>
１秒間のチック数を指定する。（デフォルト 50）

-d
executor を daemon として起動する。
-g
OzDebugf() によるメッセージ出力を行う。
-i
グローバルアクセスの表示をしない。
-p
profile を行う。
-s
手動による OZ++ System の起動を行う（shell を起動する）。
-v
executor のバージョン表示のみを行う。
-a
エグゼキュータのEXTERNシンボルを全てリンク可能にする。
-o
コンソールを開いて表示する（icon化しない）。

2.5.2 備考

SIGTERM
エグゼキュータの終了（ＯＭ経由）
SIGINT
エグゼキュータのシェル起動
SIGHUP
エグゼキュータの強制終了
