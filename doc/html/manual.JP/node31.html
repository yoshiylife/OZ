5.11 宣言

　

宣言は識別子に解釈を与える。しかし、必ずしも識別子と結び付けられる記憶 領域を確保するとは限らない。

宣言には次のものがある。

  • クラスの宣言
  • スタティッククラスの宣言
  • インスタンス変数、メソッドの宣言
  • レコードの宣言
  • メンバ、オペレータの宣言
  • 局所変数の宣言
  • 例外の宣言
  • 定数の宣言

5.11.1 クラスの宣言

クラスの宣言はクラスの定義と同時に行なわれる (5.13 節参照)。

5.11.2 スタティッククラスの宣言

スタティッククラスの宣言はスタティッククラスの定義と同時に行なわれる (5.14 節参照)。

5.11.3 インスタンス変数、メソッドの宣言

クラスおよびスタティッククラスの定義において、インスタンス変数の宣言を 行なう。インスタンス変数の宣言はインスタンス変数として識別子に型を結び付 ける (5.11.6 節参照) 。またメソッドの 宣言はメソッドの定義と同時に行なわれる (5.13.2 、 5.14.2 節参照) 。

5.11.4 レコードの宣言

レコードの宣言はレコードの定義と同時に行なわれる (5.12 節参照)。

5.11.5 メンバ、オペレータの宣言

レコードの定義において、メンバの宣言を行なう。メンバの宣言はメンバとして 識別子に型を結び付ける (5.11.6 節参 照) 。またオペレータの宣言はオペレータの定義と同時に行なわれる (5.12.1 節参照) 。

5.11.6 局所変数の宣言

　 

var_decl
        : simple_quals_type simple_vars_list ';'
        | class_quals_type class_vars_list ';'
        ; 

simple_quals_type
        : quals type
        ;

class_quals_type
        : CLASSNAME
        | 'global' CLASSNAME
        | TYPEPARAM
        ;

quals
        : /* empty */ 
        | 'unsigned'
        ;

type
        : 'char'
        | 'short'
        | 'int'
        | 'long'
        | 'float'
        | 'double'
        | 'condition'
        | 'void'
        ;

simple_vars_list
        : vars_only 
        | simple_vars_list ',' vars_only
        | exp_vars
        | simple_vars_list ',' exp_vars
        ;

vars_only 
        : vars
        ;

vars
        : processes var arrays 
        | processes arrays 
        | processes '(' vars ')' arrays 
        ;

var
        : IDENTIFIER 
        ;

arrays 
        : /* empty */ 
        | arrays array 
        ;

array
        : '[' ']' 
        ;

processes
        : /* empty */ 
        | processes process 
        ;

process
        : '@' 
        ;

exp_vars
        : vars ASSIGN exp
        | vars ASSIGN exp_with_block
        ;

vars_list_noinit
        : vars_noinit
        | vars_list_noinit ',' vars_noinit
        ;

vars_noinit
        : vars
        ;

class_vars_list
        : vars_only     
        | class_vars_list ',' vars_only 
        | exp_vars
        | class_vars_list ',' exp_vars
        | inst_init_vars
        | class_vars_list ',' inst_init_vars
        ;

inst_init_vars
        | vars '=>' IDENTIFIER '(' arg_list ')' inst_option exp_invokes
        ;

exp_invokes
        : /* empty */
        | exp_invokes exp_invoke
        ;

exp_invoke
        | '->' IDENTIFIER '(' arg_list ')' 
        ;

arg_list
        : /* empty */
        | exp_no_comma
        | exp_comma
        ;

inst_option
        : /* empty */
        | '@' exp %prec OM
        ;

exp_with_block
        : '{' exp_with_block_content '}' 
        ;

exp_with_block_content
        : exp_with_block ',' exp_with_block
        | exp_with_block ',' exp
        | exp ',' exp_with_block
        | exp
         ;

ブロックの直後に、そのブロックで有効な局所変数を宣言できる。局所変数の 宣言は識別子に型を結びつける。さらに型がクラス型の場合のみ、ローカル、 グローバルの属性が付与される。 

class A;
statci class B;

A a1;
global A a2;        // a1はローカルオブジェクト、a2はグローバルオブジェクト
global B b1;        // エラー: Bはスタティッククラス

また局所変数の初期化の記述が可能である。型がレコードの場合には、{} を用いて記述する。 

record class Point {
  int X, Y;
}

Point p1 = { 0, 0 }, p2 = { 100, 100};

また、配列およびプロセスの派生型の定義もここで行なわれる (5.7 、5.8 節参照) 。 

int a[];        // aはint型の要素を持つ配列派生型
void @b;        // bは任意の型に適合するプロセス派生型

5.11.7 例外の宣言

例外の宣言は、sharedにおいて行なわれる (5.15.2 節参照)。

5.11.8 定数の宣言

ユーザ定義の定数の宣言は、sharedおいて 行なわれる (5.15.2 節参照) 。
