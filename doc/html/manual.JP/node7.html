2.3 ランチャの説明

2.3.1 概要

LauncherはユーザがOZ++で書いたプログラムを実行するためのツールです。

2.3.2 各機能の説明

画面の説明

  1. ウインドウのタイトル
  2. メニューバー
  3. カレントプロジェクト表示ビュー
  4. object、package切替えスイッチ
  5. サブプロジェクトリスト
  6. Launchable オブジェクトまたは Launchable クラスリスト

プロジェクト

  1. 「プロジェクト」はLaunchableオブジェクトを管理するための名称空間です。 Launchableオブジェクトは、この名称空間内に名前を付けて登録します。 プロジェクトは、/（ルート）で始まる木構造を持っています。

  2. ユーザは、ルートプロジェクトの下に自由にサブプロジェクトを 作成することができます。作成方法は、前述の使用例を参照してください。

  3. Launcherはカレントプロジェクトを１つ持つことができます。 これはカレントプロジェクト表示ビューに表示されます。

  4. カレントプロジェクトのサブプロジェクトはサブプロジェクトリストに 表示されます。 サブプロジェクトをダブルクリックすることによって、 クリックされたプロジェクトをカレントプロジェクトに変更することができます。

  5. 上記以外に、プロジェクトの移動（カレントプロジェクトの変更）は ``Project'' メニューからも行なえます。 Up によって、現在のカレントプロジェクトの親プロジェクトへ移動することができま す。

パッケージ

Launcherより実行可能なプログラムはパッケージという単位で カタログというグローバルオブジェクトに登録されています。 Launcherからプログラムを実行するためには、 まずカタログからパッケージをコピーする必要があります （前述の使用例「カタログよりパッケージのインポート参照）。 パッケージリストには登録されたパッケージが表示されます。 インポートしたパッケージはカタログ上の名前がそのまま採用されるため、 同一名称のパッケージを同一プロジェクト上に２つ以上インポートすることは できません。

Launchableオブジェクト

プログラムを実行するにはLauncherに登録したパッケージを Launchableオブジェクトに 変更する必要があります。 これはパッケージリストの要素をダブルクリックすることにより 行えます（前述の使用例「オブジェクトの生成と実行」を参照）。 ここで入力するNameはLauncher上で使用される名前です。 なお、この名前は、プロジェクトの中でユニークでなければなりません。 また、同一プロジェクト内でのサブプロジェクト名、パッケージ名とも重複できません 。 生成されたLaunchableオブジェクトはLaunchableオブジェクトリストに表示され、 ダブルクリックによりそのオブジェクトが実行されます。

編集機能

サブプロジェクト、Launchableオブジェクト、パッケージの移動、 削除はすべて ``Edit'' メニューから行うことができます。

  • 移動または削除したいアイテムをマウスで選択する。
  • メニューから ``Move''、 ``Delete'' を選択する。
  • ``Move'' では、ダイヤログボックスに必要な項目を記入する。

この際、 移動の場合移動、複写先のプロジェクトは既に存在していなければなりません。

また、ルートプロジェクトは移動、削除はできません。

2.3.3 その他の機能

``System'' メニューの ``Flush'' は Launcher イメージをセーブする時に 用います。 ``Shutdown'' はエクゼキュータを終了する時に用います。

2.3.4 使い方

使用例

Launcherの基本的な機能を理解するためにLauncherの利用例を挙げます。

  1. サブプロジェクトの生成と移動
    1. ``Create'' メニューの ``Project'' を選択します。 なお、マウス操作はすべて左ボタンによって行います。

    2. この操作によってダイアログボックスが表示されます。 このダイアログボックスの入力フィールドに ``Tools'' と入力して ``Accept'' ボタンをクリックします。 サブプロジェクトリスト（左側のリストボックス）に ``Tools'' が 表示されます。

    3. サブプロジェクトリスト上の ``Tools'' をダブルクリックします。 カレントプロジェクトの表示（メニューバーの下のビュー）が ``/'' から ``/Tools'' に変わる。

  2. カタログよりパッケージのインポート
    1. ``Create'' メニューの ``Package'' を選択します。 カタログブラウザの画面が表示されます。

    2. カタログブラウザ左側のDirectoriesリストの中の ``Tools'' を ダブルクリックします。 これによって、カレントディレクトリが移動します。

    3. カタログブラウザ右側のentriesリストより ``WorkbenchLaunchable'' を シングルクリックで選択した後、下にある ``Select'' ボタンをクリックします。 Select 実行後は、カタログブラウザの画面は消えます。 もし操作を中断したい時は ``Quit'' ボタンをクリックします。 なお、カタログブラウザ表示中はLauncherの操作はできません。

      注意）entriesリストはダブルクリックはしないこと。

  3. Launchableオブジェクトの生成と実行
    1. ``package'' と表示されているラジオボタンを押下します。 これにより、右側のリストボックスがLaunchableオブジェクトリストから packageリストに切り替わります。 先ほどカタログブラウザから入力した ``WorkbenchLaunchable'' が 表示されます。

    2. リストの ``WorkbenchLaunchable'' をダブルクリックします。 ダブルクリックによって、ダイヤログボックスが表示されます。

    3. ダイアログボックスの入力フィールドに ``wb1'' と入力して ``Accept'' ボタンをクリックします。

    4. objectと表示されているラジオボタンをクリックします。 これにより、再び右側のリストボックスがLaunchableオブジェクトリストに戻ります。 リストには、 ``wb1'' が表示されています。

    5. Launchableオブジェクトリスト上の ``wb1'' をクリックします。 この操作により、wb1は起動され、ワークベンチの初期画面が表示されます。
