5.4 基本データ型

　

OZ++ 言語では基本データ型として、以下のものがある。

  • 文字 ( char)

    unsignedの指定が可能である。

  • 整数 ( short, int, long)

    整数を表現する型として、 short、 int、 long の三種類 の異なるサイズのものが利用可能である。

    それぞれに対して、 unsignedの指定が可能である。

  • 浮動小数 ( float, double)

    浮動小数を表現する型として、 float、 doubleの二種類の 異なるサイズのものが利用可能である。

  • 条件変数 ( condition)

    スレッド間の同期をとるための条件変数を表現する型である (5.19 節参照)

基本データ型は値として解釈され、後述する代入および比較は値に対して実行 される。また、代入および後述するメソッド呼び出しの引数または返り値に基 本データ型が使用された場合には、次のようになる。

  • condition 型の場合

    参照が渡される。

  • それ以外の場合

    すべて値がコピーされる。 

int a, b;

a = 2;                  // 2 が a にコピーされる 
b = a;                  // b に a の値がコピーされる 
if (a < 10)             // a の値(2) と値 10 が比較される 
  {
    math->log(2);       // a の値(2) がコピーされて仮引数に代入される 
  }

OZ++ 言語では基本データ型の消費する領域のサイズは、C や C++ のように処理系 の実装依存ではなく、言語においてすべて規定している。ただし、各基本デー タ型のアライメント等は実行環境に依存する。

5.4.2 0 (ゼロ)

　

0 は右辺値として OZ++ 言語において任意の型に対して利用可能な 値である。

その値の意味は以下のようにそれが利用される型によって決まる。 

class A;

A a;
int a = 0;     // 整数の 0 を表す。

if (a == 0)    // クラスのインスタンスの参照がないことを表す。
  ...

  • condition 型を除く基本型の場合は、その型において 0 が表現 する値を表す

  • condition 型の場合は、参照がないことを表す

  • クラスおよびスタティッククラスの場合は、インスタンスへの参照が ないことを表す (5.13、 5.14 節参照)

  • レコードの場合は、そのすべてのメンバが 0 であることを表す。 (5.12 節参照)

  • 配列の場合は、配列への参照がないことを表す (5.7 節参照)

  • プロセス型の場合は、いずれのプロセスも参照していないことを表す (5.8 節参照)
