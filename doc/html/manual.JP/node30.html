5.10 文

　 

stmt
        : comp_stmt
        | not_comp_stmt
        ;

not_comp_stmt
        : ';'
        | exp_stmt
        | select_stmt
        | iterate_stmt
        | jump_stmt
        | except_stmt
        | c_stmt
        | concurrent_stmt
        | debug_stmt
        ;

文には以下のものがある。

  • 式文
  • 複文 (ブロック)
  • 宣言文
  • 選択文
  • 反復文
  • ジャンプ文
  • 例外処理文
  • 他言語インタフェース (C 文)
  • 並行プログラミングのための文
  • デバッグ文

5.10.1 式文

exp_stmt
        : exp ';'
        ;

式を評価する文である。式文の副作用は、次の文が実行される前に完了する。 exp を省略している式文は、空文と呼ばれる。空文は反復文 (5.10.5 節参照) の本体を空にする場合などに利用さ れる。

5.10.2 宣言文

var_decls
        : /* empty */
        | var_decls var_decl 
        ;

var_decl
        : simple_quals_type simple_vars_list ';'
        | class_quals_type class_vars_list ';'
        ;

新しい識別子をブロックに取り入れることを宣言する文である。 (5.11 節参照)

宣言で取り入れた識別子が既に外側のブロックで宣言されている場合、その 外側の宣言はそのブロック内では隠されそのブロックを脱出すると再び効果を 持つようになる。

各ブロックで宣言された局所変数はブロックを脱出する時点で破壊される。

5.10.3 複文 (ブロック) 

comp_stmt
        : block
        ;

block
        : '{'   block_contents '}'
        ;

block_contents 
        : var_decls stmts
        ;

stmts
        : /* empty */
        | stmts stmt
        ;

一つの文しか使えない場所でも複数の文を使えるように、複数の文をひとまと めにする文で、ブロックとも呼ぶ。

文を一つも持たないブロックを空ブロックと呼ぶ。

5.10.4 選択文

select_stmt
        : if_exp if_body
        | 'switch' '(' exp ')' '{' label_stmt '}'
        ;

if_exp
        : 'if' '(' exp ')' 
        ;

if_body 
        : stmt 
        | stmt 'else' stmt
        ;

label_stmt
        : label not_comp_stmts
        | label_stmt label not_comp_stmts 
        ;
label
        : 'case' exp ':'
        | 'default' ':'
        ;

not_comp_stmts
        : /* empty */
        | not_comp_stmts not_comp_stmt
        ;

いくつかの制御の流れから条件に応じて一つを選択する文である。

選択文の中の stmt は、単一の宣言 (5.11 節 参照) であってはならない。これを許すと、スコープが明確でない、無駄な名 前を導入することになる。 

if (i) 
  int a = 7;        // エラー

if (i)
  {
    int a = 7;      // OK: aはブロック内で有効
  }

case ラベルの式には定数の一次 式 (5.9.1 節参照) のみが使用可能である。

5.10.5 反復文

　 

iterate_stmt
        : 'while' '(' exp ')' stmt
        | 'do' stmt 'while' '(' exp ')' ';'
        | 'for' '(' exp_or_null ';' exp_or_null ';' exp_or_null ')' stmt
        ;

exp_or_null
        : /* empty */
        | exp
        ;

繰り返しの構造を記述する文である。

反復文の中の stmt は、単一の宣言 (5.11 節 参照) であってはならない。

5.10.6 ジャンプ文

jump_stmt
        : 'break' ';'
        | 'continue' ';'
        | 'return' ';'
        | 'return' exp ';'
        ;

無条件に制御を移動するための文である。

5.10.7 例外処理文

　 

except_stmt
        : 'try' block 'except' '{' except_body default_except_block '}'
        | 'raise' exception_label2 ';'
        | 'raise' ';'
        ;

except_body
        : /* empty */
        | except_body exception_labels block 
        ;

default_except_block
        : /* empty */
        | 'default' block
        ;

exception_labels
        : exception_label 
        | exception_labels ',' exception_label 
        ;

exception_label
        : exception_name 
        | exception_name '(' IDENTIFIER ')'
        ;

exception_label2
        : exception_name
        | exception_name '(' exp ')'
        ;

exception_name
        : SHAREDNAME EXCEPTNAME
        | IDENTIFIER
        ;

例外を受けとり処理する文と例外を発生する文である。(5.16 節参照)

5.10.8 他言語インタフェース

c_stmt
        : 'inline' '"C"'  '{' inline_body '}'
        ;

inline_body
        : /* empty */
        | inline_body STRINGS
        ;

OZ++ 言語のソースコード中に C 言語のソースコードを記述する他言語 インタフェースのための文である。 (5.22 節参 照)

5.10.9 並行プログラミングのための文

　 

concurrent_stmt:
          'detach' exp ';'
        | 'kill' exp ';'
        | 'abort' ';'
        | 'abortable' ';'
        | 'wait' exp ';'
        | 'signal' exp ';'
        | 'signalall' exp ';'
        ;

これらは、 OZ++ 言語によって並行プログラミングを実現するための言語要素 の一部である。

detach、 kill、 abort、 abortable はプロセスに対す る処理 (5.8.3 節参照) を記述するため のものである。

他の三つは、スレッド間の同期のための condition 型の変数に対する 演算である (5.19 節参照) 。

5.10.10 デバッグ文 

debug_stmt:
        : 'debug' '(' exp ',' STRING ')' ';'
        | 'debug' '(' exp ',' STRING ',' exp ')' ';'
        | 'debug' '(' 'default' ',' STRING ')' ';'
        | 'debug' '(' 'default' ',' STRING ',' exp ')' ';'
        | 'debug' compound_stmt
        ;

デバッグ時の、メッセージ出力や必要な処理を記述するための文である (5.20 節参照) 。
