5.1 概要

OZ++ は、オブジェクトの交換と共有に基づくオブジェクト指向分散環境であり、 その上でのプログラミングを行なうための言語が OZ++ 言語である。 OZ++ 言語 は、ネットワーク上に分散したオブジェクトに対して位置透過なプログラミン グが可能な、以下のような特徴を持つ分散オブジェクト指向プログラミング言 語である。

  • 型システムをサポート

  • クラス名空間をサポート

  • マルチスレッドプログラミングをサポート

OZ++ 言語は 以下の点で C 言語をベースとしている。

  • 一部を除く C の構文における式 (C と同じ演算子を利用)

  • すべての制御構造

  • 基本型および基本型における型変換

一方、C 言語の以下の機能は削られている。これは、 OZ++ 言語のオブジェクト 指向言語としての性格づけを強めるとともに、誤ったポインタの使用によって エグゼキュータを破壊しそのために複数ユーザによって共有されるオブジェク トが利用不能となる事態を回避し、システム全体を安定して動作させるためで ある。

  • 型システムの一部 (構造体、共用体、ポインタ型、列挙型、型定義など)

  • 型定義

  • ユーザ定義の関数

ただし、既存の C 言語のライブラリ等の資産を利用したり、低レベルのプロ グラミングを行なうために、いくつかの制約のもとで C 言語による記述が可 能な他言語インタフェースを用意している。

また、分散オブジェクト指向プログラミング言語とするために、以下のような C 言語にはない要素を追加している。

  • 三つの利用者定義型 (クラス、スタティッククラス、レコード)

  • メッセージ式

  • 並行プログラミングのための言語要素

  • 例外処理

この利用者定義型の中のクラスとスタティッククラス、およびメッセージ 式によってオブジェクト指向の要素を追加している。クラスでは多重継承が可能 で、メソッドは動的束縛である。スタティッククラスでは継承および メソッドの動的束縛を排除することによって、粒度の小さいオブジェクトを効 率良く実装することを可能にする。

また OZ++ においてはオブジェクトをグローバルオブジェクトとローカルオブ ジェクトの二種類のもので提供する。

  • グローバルオブジェクト

    分散透過にアクセスされるオブジェクト

  • ローカルオブジェクト

    グローバルオブジェクトの部品となるオブジェクト

OZ++ では、異なる計算機上に存在するグローバルオブジェクト 間のメッセージ式を記述することによって、それらの間でネットワークを介し た情報のやりとりを行なう分散プログラミングのが可能である。
