5.23 メソッド実行

　

ここでは、 OZ++ 言語のクラス、スタティッククラスのメソッド、およびレコー ドのオペレータの実行時の動作について述べる (5.13.2、 5.14.2、 5.12.1 節参照) 。

5.23.1 クラスのメソッド

クラスのメソッドは、private のものとそれ以外のものとで実行時の動作 が異なる。

  • private メソッド

    private メソッドはそれが定義されたクラス中で のみ利用されるため、コンパイル時に静的に実行コードが決定される。したがって、 ソースコード中のメソッド起動は、コンパイル時に 単なる関数呼び出しの形式に変換される。

  • private 以外のメソッド

    private 以外のメソッドは、対応する実行コードが動的にロードされ、その ロードされたものが実行される。したがって、ソースコード中のメソッド起動 は、実行時にロードされる実行コードの中でそのメソッドに対応するものが呼 び出されるように変換されなければならない。

    また動的束縛が行なわれるため、変数の型ではなく、その変数が参照している インスタンスの型に応じて実行されるメソッドが決定される。さらに、グロー バルメソッドの起動の場合には、引数のローカルオブジェクトのコピー等の特 別な処理が必要である。

    このように private 以外のメソッド起動では、単純な関数呼び出しではなく、 エグゼキュータのランタイムルーチンを利用した特殊な処理が行なわれる。

5.23.2 スタティッククラスのメソッド、オペレータ

スタティッククラスのメソッドおよびレコードのオペレータは、アクセス制御 の指定に無関係にすべて単なる関数呼び出しとして実行される。

ところで public のメソッドは、すべて他のクラス等で利用されるため、それ らのコンパイル後の関数の名前は OZ++ システム全体でユニークでなければな らない。コンパイラでは、これらの関数名にそれが定義されているスタティッ ククラスまたはレコードの ID を含めることによってこれを保証している。
